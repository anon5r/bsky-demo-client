# ⚠️ [廃止] サードパーティクライアント認証ガイド

---

## 🚫 このドキュメントは廃止されました（DEPRECATED）

**このドキュメントは古い設計に基づいており、現在の実装では使用されていません。**

**⚠️ WARNING:**
このドキュメントの内容に従って実装しないでください。最新のドキュメントを参照してください。

### 廃止理由

このドキュメントは古い OAuth 認可サーバーアプローチに基づいており、以下の問題がありました：

- Chronosky が独自の OAuth 認可サーバーとして動作する設計（AT Protocol 非準拠）
- サードパーティクライアントに Chronosky 専用トークンの取得を要求
- Bluesky で既に取得済みのトークンが使用できない
- クライアント事前登録が必要

これらの問題を解決するため、2026-01-12 に AT
Protocol 準拠の新設計に移行しました。

## 主な変更点

### 旧設計（このドキュメント）

- Chronosky が OAuth 認可サーバーとして動作
- サードパーティクライアントが Chronosky 専用のトークンを取得
- 既存の Bluesky トークンは使用不可

### 現在の設計（2026-01-12 以降）

- **AT Protocol 準拠のオープンアクセスモデル**
- Bluesky OAuth トークンをそのまま使用
- トークン交換不要
- クライアント事前登録不要

## 移行ガイド

既にこのドキュメントに基づいて実装している場合は、以下の手順で移行してください：

1. **Chronosky 専用の OAuth フローを削除**
   - Chronosky 用のトークン取得・管理コードを削除
   - クライアントメタデータの公開を停止

2. **Bluesky トークンを直接使用**
   - 既存の Bluesky OAuth で取得したトークンを使用
   - DPoP proof の生成は同じ

3. **API エンドポイントは変更なし**
   - XRPC メソッド名は同じ
   - リクエスト形式も同じ

詳細は [最新ドキュメント](../THIRD_PARTY_CLIENT_GUIDE.md) を参照してください。
